# Project Intent

## 목적

2026년 7월 28일 일본 키타노역 근처 자취 시작을 준비하기 위한 생활필수품 검색 허브다. 사용자는 일본 생활 초반에 필요한 물건을 빠르게 파악하고, 온라인 주문 또는 하치오지역 주변 오프라인 매장 검색으로 바로 이어가고 싶어 한다.

## 핵심 사용자 상황

- 입주 직후 당장 필요한 물건을 빠르게 우선순위화해야 한다.
- 일본어 상품명을 매번 직접 생각해서 검색하기 번거롭다.
- 라쿠텐, Amazon JP, 니토리, 빅카메라 같은 유명 일본 사이트의 검색 링크가 한곳에 모여 있으면 좋다.
- 출퇴근을 하게 될 경우 하치오지역 근처에서 바로 살 수 있는 생활품 매장도 확인해야 한다.
- 지도 검색도 같이 필요하다.

## 현재 제품 방향

이 사이트는 상품 데이터를 직접 크롤링하지 않는다. 대신 품목별 일본어 검색어를 관리하고, 각 쇼핑몰/지도 검색 URL을 안정적으로 생성한다.

이 선택의 이유:

- 라쿠텐, Amazon, 지도, 공식몰 검색 결과는 자주 바뀐다.
- 직접 크롤링은 차단, 약관, 유지보수 문제가 생기기 쉽다.
- 사용자가 실제 구매 전 최신 결과를 각 사이트에서 직접 확인하는 편이 안전하다.
- 정적 사이트로 유지하면 Vercel 배포와 GitHub 관리가 단순하다.

## 주요 기능

- 상단 탭 구조(8개): `체크리스트` / 공간별 킷 5개(`주방·소품`·`거실`·`욕실·세탁`·`침실·수납`·`업무·출퇴근`) / `한국 식품·뷰티` / `지도`
- 기준 위치 선택(키타노역·하치오지역) — 검색 링크와 지도에 공통 적용
- 상황 필터: 전체, 도착 당일, 첫 주, 재택/업무, 출퇴근 대비
- 빠른 텍스트 필터
- D-day 카운트다운(입주일 자동 계산) + 단일 상태 바(진행률·남은 예산 — 중복 요약 카드 없음)
- 생활필수품 체크리스트: 우선순위 그룹 컨테이너 + 컴팩트 리스트 행(행 펼침으로 노트·검색어·전체 몰 링크, 접힌 상태에선 대표몰 검색 버튼 1개만)
- 그룹별 미체크 소계 + 전체 남은 예산(미체크 항목 기준, 체크 즉시 갱신)
- 품목별 베스트 픽(`pick` 필드): 라쿠텐 리뷰순 실검색으로 검증한 추천 상품 1개를 행 안에 별도 버튼(★ 추천)과 펼침 상세 박스(평점·리뷰수·가격·검증일)로 노출. **픽은 수동 큐레이션**이며 갱신 시 검증일("YYYY-MM 확인")을 함께 바꾼다. minBudget도 같은 시점의 실가격 기준.
- 카테고리별 추천 스토어(`recommendedShops`): 리뷰가 많은 유명 라쿠텐 전문 스토어의 **스토어 내 검색** 링크(`search/mall/{키워드}/?sid={스토어ID}`). 펼침 상세에 "추천 스토어 / 전체 몰 검색" 두 줄로 구분. sid는 각 스토어 톱페이지의 검색 폼에서 추출해 검증한다. 뷰티는 Qoo10 OLIVE YOUNG 공식몰 고정 링크.
- 대표(첫 번째) 검색 버튼은 카테고리별로 다르다(`PRIMARY_SHOP_BY_CATEGORY`): 가전=요도바시(무료배송·설치), 주방·생활·출퇴근=Amazon(빠른 배송·한국어 UI), 가구·한국식품=라쿠텐(전문 스토어), 뷰티=Qoo10. 사용자가 해당 몰을 끄면 표시 중인 첫 몰로 폴백.
- 한국 식품·뷰티 탭: 한식 조미료·김치·라면·K뷰티 18품목, Qoo10 마켓 포함(식품·뷰티에는 메루카리 제외). 오프라인은 業務スーパー 하치오지점(키타노역 도보 약 10분)·韓美膳DELI 세레오하치오지.
- 체크 상태 `localStorage` 저장(`kitanoChecks`)
- 검색 옵션 패널: 표시할 쇼핑몰 on/off, 예산 상한, 정렬, 送料無料 → 링크에 즉시 반영, `localStorage` 저장(`kitanoOpts`)
- 품목별 사이즈 칩(가구류) → 검색 키워드에 반영, `localStorage` 저장(`kitanoSizes`)
- 카테고리별 쇼핑몰 노출(공통몰 + 카테고리 전문몰)
- 지도 탭: 키 없는 Google Maps 임베드(`output=embed`) + 매장/카테고리/위치 연동 이동

## 우선 품목

- 침구: 매트리스(퀸), 침대 프레임/DIY 받침, 침구 세트(이불·베개·시트)
- 가구: 소파(콤팩트), 식사용 책상·의자, 업무용 책상·의자, 수납 선반/서랍
- 대형가전: 세탁기, 냉장고
- 소형가전: 전자레인지, 밥솥, 전기포트, 청소기, 조명(실링라이트), 헤어드라이어, 서큘레이터/선풍기
- 주방: 냄비/프라이팬, 칼/도마/조리도구, 식기/수저/컵
- 생활: 청소/세탁 소모품, 커튼, 쓰레기통(분리수거)
- 출퇴근 준비품

## 유지보수 기준

- 새 품목은 `app.js`의 `items` 배열에 추가한다. 필드: `base`(일본어 검색어), `category`(furniture|appliance|kitchen|supplies|commute), 필요 시 `sizes`+`defaultSize`(사이즈 칩), `priority`, `minBudget`, `mode`, `note`.
- 새 매장은 `app.js`의 `stores` 배열에 추가한다. 필드: `name`, `cat`(지도 카테고리 칩), `tags`, `query`.
- 새 쇼핑몰은 `marketplaces` 배열(객체: `id`,`name`,`appliesTo`)에 추가하고, `searchUrl()`의 `switch`에 해당 `id`의 검색 URL 생성 분기를 추가한다. `appliesTo`는 `"all"` 또는 카테고리 배열.
- 가격/정렬 파라미터는 브라우저에서 실제 URL로 검증된 것만 적용한다. 현재 검증됨: 가격 상한 = Rakuten `max`·Amazon `high-price`·Yahoo `pt`·메루카리 `price_max`, 정렬 = Rakuten `s=2`(가격)/`s=5`(리뷰)·Amazon `s=`·메루카리 `sort/order`. 사이즈·`送料無料` 같은 조건은 검색 키워드에 붙여 모든 몰에 견고하게 반영한다.
- 오프라인 매장은 실존이 웹으로 검증된 곳만 `stores`에 넣는다(공식 매장 페이지 확인). `mapQuery`는 키 없는 임베드(iframe)에서 **핀+장소카드가 실제로 뜨는지** 확인한 값만 쓴다 — 임베드의 장소 검색은 본 지도보다 약해서, 구글맵에 POI가 없거나 이름 매칭이 불안정한 매장(예: 韓美膳DELI, ウエルシア역북구점)은 **번지 주소 문자열**(예: `東京都八王子市東町7-6`)로 핀을 잡고 tags에 층·건물 안내를 적는다. 디자인 원칙: 카드 남발 금지 — 품목·매장은 그룹 컨테이너 안의 리스트 행, 그림자는 지도/스티키 바 정도에만.
- 외부 API 키가 필요한 기능은 기본값으로 넣지 않는다. 지도는 키 없는 임베드 URL만 사용한다.
- 사이트 첫 화면은 체크리스트와 검색 행동이 바로 보이게 유지한다.
- README는 실행/배포용, 이 파일은 의도/맥락용으로 분리한다.

## 작업 방식 (역할 분담)

- 기획, 계획, 지시(스펙 작성), 검토는 Opus(메인 에이전트)가 담당한다.
- 실제 코딩(파일 수정/구현)은 Sonnet 서브에이전트가 담당한다.
- Opus는 구체적인 스펙을 전달하고, 구현 결과를 브라우저에서 직접 검증한 뒤 커밋한다.
